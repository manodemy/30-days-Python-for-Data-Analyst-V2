The Complete 20-Day SQL Curriculum

Every concept a professional Data Analyst uses on the job — in a logical, progressive sequence.

Phase 1 Foundations — Days 1–5 📦 retail.db
  • Day 01
    Introduction to SQL & Databases
    SELECT *, SELECT columns, column aliases (AS), DISTINCT, what is a database?
    🎯 List all unique product categories sold
  • Day 02
    Filtering Data with WHERE
    WHERE with comparison ops, AND/OR/NOT, BETWEEN, IN/NOT IN, operator precedence
    🎯 Find all sales over ₹5,000 in the "North" region
  • Day 03
    Pattern Matching & NULL Handling
    LIKE with % and _, IS NULL / IS NOT NULL, why NULL ≠ empty string, COALESCE
    🎯 Find customers whose names start with 'A' or have NULL email
  • Day 04
    Sorting & Limiting Results
    ORDER BY (ASC/DESC), multi-column sort, LIMIT, OFFSET, Top-N queries, NULLS FIRST/LAST
    🎯 Get the top 10 highest-value sales, most recent first
  • Day 05
    Aggregate Functions
    COUNT(*) vs COUNT(col), SUM, AVG, MIN, MAX, ROUND, aggregates with WHERE
    🎯 What is the average sale value per category?
Phase 2 Grouping & Shaping Data — Days 6–8 📦 retail.db (extended)
  • Day 06
    GROUP BY & HAVING
    GROUP BY single/multiple columns, HAVING (post-aggregation filter), frequency analysis patterns
    🎯 Find categories with 100+ orders AND avg sale > ₹3,000
  • Day 07
    Data Types, Casting & Expressions
    CAST, arithmetic in SELECT, ROUND/ABS/MOD, string concatenation (||), type traps with dates
    🎯 Create a "revenue_tier" column based on amount ranges
  • Day 08
    CASE WHEN — Conditional Logic
    Simple & searched CASE, CASE inside GROUP BY, conditional aggregation (the pivot trick)
    🎯 Classify each sale as Low/Medium/High; count each tier
Phase 3 Joins & Relationships — Days 9–11 📦 company.db
  • Day 09
    Understanding Relationships & INNER JOIN
    Foreign keys, primary keys, reading ERDs, INNER JOIN, JOIN ON vs USING, multi-table joins
    🎯 Get employee names with their department and salary
  • Day 10
    LEFT, RIGHT & FULL OUTER JOIN ⭐ Mini Project
    LEFT JOIN, RIGHT JOIN, FULL OUTER JOIN, finding unmatched rows pattern, CROSS JOIN
    🎯 Find all departments with NO employees assigned
  • Day 11
    SELF JOIN & Complex Multi-Table Queries
    SELF JOIN for hierarchies, table aliasing, troubleshooting duplicate rows in JOINs
    🎯 List every employee with their manager's name
Phase 4 Advanced Querying — Days 12–15 📦 ecommerce.db
  • Day 12
    Subqueries
    Scalar subqueries, IN/NOT IN, EXISTS/NOT EXISTS (correlated), subquery in FROM (derived tables)
    🎯 Find customers who ordered more than the average order value
  • Day 13
    CTEs (Common Table Expressions)
    WITH clause, chaining CTEs, recursive CTE intro (org charts), CTE vs temp table
    🎯 Use a CTE to rank products by revenue; filter top 5 per category
  • Day 14
    Window Functions Part 1 — Ranking
    OVER(), PARTITION BY, ROW_NUMBER(), RANK(), DENSE_RANK(), NTILE(n)
    🎯 Rank customers by total spend within each region
  • Day 15
    Window Functions Part 2 — Analytic
    LAG(), LEAD(), FIRST_VALUE(), LAST_VALUE(), running totals, moving averages
    🎯 Calculate month-over-month revenue growth % using LAG
Phase 5 Real-World Skills — Days 16–19 📦 mixed datasets
  • Day 16
    String Functions
    UPPER/LOWER, TRIM/LTRIM/RTRIM, LENGTH, SUBSTR, REPLACE, INSTR, CONCAT — real data cleaning patterns
    🎯 Standardize a messy customer names column
  • Day 17
    Date & Time Functions
    STRFTIME formatting, date extraction, date arithmetic, JULIANDAY, "last 30 days" pattern
    🎯 Find all orders placed on weekends in Q4
  • Day 18
    UNION, INTERSECT & EXCEPT (SET Operations)
    UNION vs UNION ALL, INTERSECT, EXCEPT, column matching rules, stacking monthly reports
    🎯 Stack 2023 and 2024 sales; find customers who bought in both years
  • Day 19
    Query Optimization & Best Practices
    EXPLAIN QUERY PLAN, index basics, sargable predicates, avoiding SELECT *, query formatting
    🎯 Identify and rewrite a slow, poorly-formatted query
Capstone BI Report Capstone — Day 20 📦 capstone_retail.db
  • Day 20
    🏆 Retail Business Intelligence Report
    Write 8 production-ready SQL queries answering real business questions: MoM revenue growth, top products, customer segmentation, regional performance, return rates, sales rep ranking, customer cohort LTV, day-of-week analysis.
    🎯 Final output: a clean, commented SQL script — interview-ready
